# Mobile Tavern 预设 (Preset) Prompt 上下文分区配置指南
*Version: 1.3.5*

在“控制面板 -> 预设”设置中，Prompt 的结构化分割（Partitions）是本软件兼容 SillyTavern 提示词拼装设计的核心。它将角色设定、世界设定、破限词和纪律要求，按合理的顺序合并，注入给 AI 大语言模型。

---

## 1. 核心 Prompt 上下文分区详解

以下为预设模块中的所有分区（按生成上下文时的推荐逻辑顺序从上至下排列）：

| 分区标识 / Key | 对应标头名称 (Section Header) | 作用与注入内容说明 |
| :--- | :--- | :--- |
| **system** | 设定基础标头 (World Lore) | **系统底层扮演指令**。通常为最顶部的设定，告诉 AI 这是一个酒馆角色扮演环境，要求其遵守的默认语气、代词、文本排版与对话模式。 |
| **beforeChar** | 背景设定前置标头 | **角色前置背景设定**。用来放置大型世界观规则、社会背景或历史事件，它们在角色性格之前发挥作用，构筑宏观舞台。 |
| **personality** | 角色性格标头 | **角色性格特征 (Personality)**。对应角色卡中的 `Personality` 字段。定义角色的性格标签、情绪基调、习惯动作、口癖，是角色特异性的精髓。 |
| **description** | 角色详细描述标头 | **角色详细描述 (Description)**。对应角色卡中的 `Description` 字段。包括角色的生理外貌（身高、眼睛、装束）、生平履历、人脉关系等细节。 |
| **scenario** | 场景设定标头 | **当前故事场景 (Scenario)**。对应角色卡中的 `Scenario` 字段。告知 AI 双方开始互动时的环境位置、天气时间以及冲突契机。 |
| **summary** | 年表记忆汇总标头 | **中远期大纲提取 (Summary)**。由上下文缓冲系统提取的前情要点总结。当聊天记录超出上下文长度时，这一栏能帮助 AI 继承“之前的剧情概要”。 |
| **userPersona** | 玩家人设描述标头 | **玩家人设设定 (User Persona)**。用来向 AI 介绍玩家（`{{user}}`）的外貌、性格、对角色的态度，使 AI 在说话时能够有针对性地对玩家人设做出回应。 |
| **charSystem** | 角色附加限制标头 | **角色专属特殊约束**。部分复杂角色卡在扩展字段中声明的额外约束，常用于极其个性化的特种卡。 |
| **worldInfo** | 设定拓展 (World Info) 标头 | **世界设定集 / 世界书 (Lorebook)**。在聊天过程中，如果触发了某些关键词，对应的设定条目就会在这里按顺序渲染，向 AI 提供即时知识库支持。 |
| **beforeLast** | 临时触发规则标头 | **临近历史的动态指令**。排在聊天记录之前的最终规则，用来注入一些高优先级但临时的情境设定或道具触发规则。 |
| **jailbreak** | 破限提示词 (Jailbreak) 标头 | **写实限制解锁提示词**。用于绕过云端大语言模型自带的审查性说教，鼓励模型生成更具戏剧冲突、情感起伏以及细节丰富的成人或写实故事。 |
| **postHistory** | 生成纪律提醒标头 | **尾部纪律指令 (Post-Instruction)**。注入在最近对话记录的最末端。利用大模型的“近因效应”（最关注最后听到的指令），在这里强调最核心的生成戒律（如：绝对禁止代操玩家、对话与动作分色等）。 |

## 2. 什么是分区标头 (Section Headers)？

在预设配置的文本框中，您会看到如 `=== 角色性格设定 ===` 或 `=== 破限提示词 ===` 等占位文本。

### 它们的作用：
1. **指示大模型进行语义分类**：在超长文本中，大模型容易混淆“什么是常识”、“什么是要求”、“什么是人设”。使用特殊的 Markdown 符号包裹的标头（如 `=== 标题 ===`），能够明确提示 AI 接下来这一段文本是特定类型的人设块。
2. **隐藏标头节省 Token**：如果在输入框中将某个标头的文本**完全清空**，拼装引擎就会在生成 Prompt 时直接略过这个标头的标题，直接拼装内容，从而进一步压缩上下文长度或隐藏段落层级。

---

## 3. 分区的拼装优先级与最佳实践

AI 的记忆和专注力会随着上下文拉长而产生漂移。通常，模型对 **最顶部 (System)** 和 **最底部 (Post-Instruction / Post History)** 的指令最为敏感。

### 调校建议：
* **破限词 (Jailbreak) 与 纪律词 (Post History)**：如果您发现模型容易出戏、或者经常开始“代操玩家 (`{{user}}`)”的角色说话，应当将限制代操的指令写在 `postHistory`（生成纪律提醒标头）中。
* **背景设定 (Lorebook/World Info)**：不需要一开始就塞满所有世界设定。通过词条匹配（例如聊到 `帝国` 才激活 `帝国` 词条）注入到 `worldInfo` 分区中，是节省 token、防止大模型脑载过载的最有效方式。

### 关键结构说明

| 字段 | 说明 |
| :--- | :--- |
| 第 1 个 `system` 消息 | 包含 `system` → `beforeChar` → `personality` → `description` → `scenario` → `summary` → `userPersona` 所有静态分区，合并为一条消息 |
| 第 2 个 `system` 消息 | `worldInfo` 分区独立注入（仅在关键词"加密文件"被触发时才出现） |
| 中间的 `user` / `assistant` 交替 | 最近 N 轮对话历史（由"上下文发送轮次"参数控制） |
| 倒数第 2 个 `system` 消息 | `beforeLast`（临时触发规则） + `jailbreak`（破限词）紧贴最后一条历史之前注入 |
| 最后 1 个 `system` 消息 | `postHistory`（生成纪律提醒）位于整个 messages 数组最末尾，利用近因效应最大化约束力 |
