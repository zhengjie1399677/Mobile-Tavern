## 0. 多环境测试策略说明（重要前置声明）

### 0.1 项目物理部署形态

本项目 **Mobile Tavern** 是基于 Tauri 构建的纯移动端原生 App（当前为 Android APK，未来扩展至 iOS），遵循 AGENTS.md 核心行为准则三「纯移动端战略与原生适配规范」。

**物理实际**：
- 客户端为单机离线运行 App，所有核心业务逻辑（Kernel / Services / Middlewares / Pipeline）打包进 APK 内的 WebView 资源
- 服务端仅在本地开发环境以 Express 形态存在（`server.ts`），用于开发调试；打包发布时**不**进入 APK
- 项目不存在传统 Web 项目的「QA 服务器集群 / Staging 集群 / Production 集群」物理分离部署架构

### 0.2 四环境在本项目中的对应映射

为严格遵循用户「必须在 Dev / QA / Staging / Production 四环境执行测试」的要求，按移动端 App 工程实践将四环境映射如下：

| 环境名称 | 本项目物理对应 | 测试载体 |
|---------|--------------|---------|
| **Development（开发环境）** | Windows 11 + Node.js 22 + TypeScript 编译时类型检查 | `npm run lint`、`tsc --noEmit` |
| **Testing/QA（测试环境）** | Node.js 22 运行时 + Vitest/自研测试Runner | `npm run test` 全量单元测试 / 集成测试 |
| **Staging（预生产环境）** | Tauri Dev 模式（`npx tauri dev`）+ Android 模拟器/真机调试 | 静态资源本地化 + 内核服务装配契约验证 |
| **Production（生产环境）** | Android 12+ 真机 + 最终签名 APK 运行环境 | 代码静态审计 + 兼容性灰度评估 |

### 0.3 测试执行原则

按 AGENTS.md 核心行为准则四「受控浏览器自动化测试规范」：
- **白名单允许**：声明式测试脚本（`tests/run_all_tests.ts`）、`npm run test` / `npm run lint` 聚合执行
- **黑名单禁止**：临时探索性浏览器调试、未固化的 E2E 冒烟
- **本地静态化**：所有测试使用本地静态资源，严禁加载境外 CDN
- **超时上限**：单测断言 ≤5s，整体测试套件无界等待防护
