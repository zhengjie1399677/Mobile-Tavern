第一步：配置“大脑”（接入 API）

推荐： 新手推荐使用 DeepSeek（因为便宜又足够好用）。
  1. 获取密钥：前往 DeepSeek 开放平台 注册账号，创建一个 API Key（以 sk- 开头的一串字符）。
  2. 填入软件：
    • 打开 APP，点击底部最右侧的 【系统设置】。
    • 找到 【接口配置】，接口类型选择 【OpenAI 兼容接口】。
    • Base URL 填入：https://api.deepseek.com/v1
    • API Key 填入你刚才复制的密钥。
  3. 连接测试：点击 【获取模型列表】，选择 deepseek-chat。然后点击 【测试连接】，提示成功即可！

第二步：注入“灵魂”（导入角色卡）

生态兼容： 兼容 SillyTavern 生态（V2 Spec）。
  1. 找卡：前往 Chub.ai 等网站，下载你喜欢的角色卡（必须是 .png 图片格式 或 .json 文本格式）。
  2. 导入：打开 APP，点击底部的 【角色名录】 -> 【导入角色卡】，选择你下载的文件。
  3. 开聊：点击刚导入的角色，进入 【聊天对话】 界面，发送你的第一句话吧！

第三步：数据防丢失

数据备份（极其重要）：
你的所有聊天记录和角色卡都物理保存在手机本地。卸载 APP 或换手机会导致数据彻底清空！请务必在 【系统设置】->【数据管理】 中，定期点击 【导出备份】 并妥善保存文件。

第二部分：详细配置指南（进阶玩家参考）

以下内容包含软件的底层逻辑与高级配置说明，供有深度定制需求的用户参考。

1. 界面模块导览

客户端主界面底部设有六个核心功能选项卡（Tabs）：

  • 角色名录 (Characters)：管理已导入的角色卡，查看卡片详情，发起新会话。
  • 聊天对话 (Chat)：当前正在进行的扮演对话视窗，支持流式文本渲染、历史消息编辑、宇宙分支建立等。
  • 会话历史 (History)：查看当前选中角色下的所有历史会话分支，支持重命名、克隆或删除分支。
  • 全局词库 (Worldbook)：配置不受角色限制的全局背景设定条目。
  • 演练场 (Playground)：临时单次测试提示词与模型的沙盒环境，不写入本地数据库。
  • 系统设置 (Settings)：配置 API 参数、采样预设（Sampler Preset）、系统默认提示词（System Prompt）以及执行数据备份。

2. 角色卡导入与管理

本软件完全兼容 SillyTavern 的角色卡生态规范。

  • 导入角色卡：在“角色名录”页面，点击“导入角色卡”按钮，选择符合标准的 .png 立绘卡片或 .json 文本文件。
  • 元数据提取：系统将自动提取以下元数据并存入本地 IndexedDB 数据库：角色名称、立绘图像、性格与设定描述、时代背景与场景设定、第一句开场白、对话示例、切换表情立绘。
  • 表情动态映射：若角色卡元数据中包含表情触发映射规则（Expression Rules），系统会在聊天过程中通过正则表达式自动匹配 AI 回复中的情绪关键词，实时将立绘图像切换为对应的表情。若无匹配规则，系统将自动降级使用卡片主立绘作为背景。

3. API 接口与模型配置

由于软件为本地离线容器，用户必须自行配置大模型 API 接口方可进行对话。

A. 使用公网商业 API 中转

  1. 进入“系统设置 -> 接口配置”区域。
  2. API 接口类型：选择“OpenAI 兼容接口”。
  3. 接口基础 URL (Base URL)：输入服务商提供的 API 根地址（如 https://api.deepseek.com/v1）。
  4. API 密钥 (API Key)：粘贴您的身份鉴权 Bearer 密钥（sk-...）。
  5. 获取可用模型：点击“获取模型列表”，在下拉菜单中选择您需要使用的模型。
  6. 连接测试：点击“测试连接”，若提示“连接成功”，即代表接口已通。

B. 使用本地离线模型（如 Ollama）

  1. 确保您的电脑与运行本 App 的手机处于同一个局域网（WiFi）下。
  2. 获取您电脑在局域网中的 IPv4 地址（Windows 用户可在 CMD 运行 ipconfig 查询）。
  3. Base URL 填入本地模型的局域网监听地址（如 Ollama 示例：http://192.168.1.100:11434）。
  4. 重要安全限制 (CORS 跨域)：由于手机 WebView 浏览器的同源安全策略，本地模型在启动时必须配置允许跨域放行。在电脑系统环境变量中添加 OLLAMA_ORIGINS=*，保存并彻底重启 Ollama 进程。

4. 世界设定集（Worldbook / Lorebook）配置与使用

设定集用于解决大模型长文本记忆力有限的问题，通过关键字匹配动态注入设定。

  • 角色专属词库：绑定在特定角色卡内的 Lorebook。仅在该角色被激活对话时才会扫描匹配。
  • 全局共享词库：在“全局词库” Tab 下配置。不论与哪一个角色对话，都会统一进行扫描匹配。
  • 激活逻辑：每个词条包含“关键字”和“描述内容”。当消息或对话中包含了配置的任意一个“关键字”，系统会自动将该关键字对应的内容临时注入到上下文指定位置；未提到关键字 of 词条将保持静默。

5. 数据导出备份与覆盖还原

数据均物理保存在手机本地的 IndexedDB 中。

  • 导出备份 (Export)：前往“设置 -> 数据管理”。支持加密导出（使用 AES-GCM 算法加密包装）和明文导出（自动抹除配置中的 API Key）。文件将写入手机存储的 /Download 公共目录下。
  • 导入还原 (Import)：点击“导入本地数据备份”，在文件管理器中选择您之前导出的备份 .json 文件。注意：数据覆盖还原将清空当前设备上原有的全部数据。
