缺陷详情列表
缺陷 #1：桥接测试脚本路径解析失败 严重 Major
npm run test:bridge 执行失败。tests/run_bridge_tests.cjs 生成的临时文件 temp_bridge.ts 中，../kernel/Kernel 导入路径无法被 Node ESM 解析器定位，报错 ERR_MODULE_NOT_FOUND: Cannot find module 'D:\projects\Mobile-Tavern\kernel\Kernel'。
位置：tests/run_bridge_tests.cjs:43 | 影响：桥接测试套件完全无法执行 | 根因：路径替换正则未覆盖 ../kernel/Kernel 导入
缺陷 #2：Markdown 表格 HTML 转换未转义单元格内容 一般 Minor
convertMarkdownTablesToHtml 函数将表格单元格内容 h 和 cellVal 直接拼接到 HTML 字符串中，未进行 HTML 实体转义。若 AI 生成内容含 <script> 标签，存在潜在 XSS 风险。实际影响有限：该输出后续经 FormattedText 自定义白名单解析器过滤，且 HTML 代码块走 iframe 沙盒。
位置：src/components/formattedTextUtils.ts:113-130 | 影响：潜在 XSS（已有缓解层） | 建议：增加 escapeHtml 转义
缺陷 #3：构建产物体积过大 建议 Suggestion
生产构建产生 3 个超过 800KB 的 chunk：builtInCharacters (2435KB)、index (2058KB)、mvu-vendor (1000KB)。移动端首屏加载将受影响，尤其内置角色卡 chunk 达 2.4MB。
位置：vite.config.ts | 建议：内置角色卡改用动态 import 懒加载，MVU vendor 拆分
缺陷 #4：动态导入与静态导入冲突 建议 Suggestion
src/hooks/useChat/helpers/index.ts 同时被 useSessionManager.ts 动态导入，又被 useChat.tsx、useRerollMessage.ts 等静态导入，导致 Vite 警告动态导入失效，模块无法被拆分到独立 chunk。
位置：src/hooks/useChat/ | 建议：统一为静态导入或重构为独立模块
缺陷 #5：esbuild CJS 格式下 import.meta 不可用 建议 Suggestion
npm run build 中 esbuild 打包 server.ts 时警告："import.meta" is not available with the "cjs" output format。当前用 __filename 兜底降级，功能正常但属于潜在隐患。
位置：server.ts:11 | 建议：将 server 输出格式改为 esm 或移除 import.meta 依赖

在 convertMarkdownTablesToHtml 中对 h 和 cellVal 调用 escapeHtml，防御性编程。
