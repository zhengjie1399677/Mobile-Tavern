## 一、评估背景与目标

### 1.1 用户需求

用户明确指令如下：

1. **仅升级安卓最低支持版本**：从 Android 7 (API 24) 提升至 Android 12 (API 31)

5. **书面确认前置**：所有代码修改必须在获得明确书面确认后才能启动

### 1.2 评估目标

## 三、Android 12 (API 31) 技术特性分析

### 3.1 Android WebView 版本独立性

**核心事实**：Android System WebView 是独立于 Android 操作系统版本的应用，通过 Google Play Store 单独更新。

| Android 系统版本         | 出厂 WebView 版本 | 实际 WebView 版本（2026年） |
| -------------------- | ------------- | -------------------- |
| Android 7.0 (API 24) | Chrome 51     | 已停止更新（最高约 100）       |
| Android 12 (API 31)  | Chrome 88-96  | 通常已更新至 120+          |
| Android 13 (API 33)  | Chrome 108+   | 130+                 |

**重要结论**：minSdk 从 24 升级到 31，**不会降低** WebView 的实际版本范围。相反，Android 12 设备的 WebView 普遍比 Android 7 设备更新，对现代 CSS/JS 特性的支持**更好**而非更差。

### 3.2 Android 12 原生 API 新增特性

Android 12 (API 31) 相比 Android 7 (API 24) 新增的关键原生 API：

| API                           | 引入版本                | 项目是否使用    |
| ----------------------------- | ------------------- | --------- |
| `MediaStore.Downloads`        | API 29 (Android 10) | ✅ 是（文件保存） |
| Scoped Storage 强制启用           | API 29 (Android 10) | ✅ 是（已适配）  |
| `enableOnBackInvokedCallback` | API 33 (Android 13) | ✅ 是（已声明）  |
| 预测式返回手势                       | API 33 (Android 13) | ✅ 是（已声明）  |

## 五、潜在风险评估

### 5.1 风险等级定义

* 🔴 **高风险**：可能导致功能不可用或崩溃

* 🟡 **中风险**：可能导致视觉异常但功能可用

* 🟢 **低风险**：几乎无影响或已有充分兜底

### 5.2 风险清单

#### 风险 R-01：oklch() 主题色在旧 WebView 上失效 🟡

**风险描述**：`oklch()` 需要 WebView 111+。若设备 WebView 未更新至 111+，主题色变量将失效，导致界面背景/前景色回退到浏览器默认值（通常为黑底白字或白底黑字）。

**影响范围**：约 60 处 CSS 变量定义（4 个主题 + 默认主题）

**触发条件**：

* 设备无 Google Play 服务（如部分华为设备）

* 用户禁用了 WebView 自动更新

* 设备厂商未推送 WebView 更新

**当前处理**：用户明确要求**不降级**，因此不提供 hex 降级值

**风险评估**：

* 在 minSdk=24 时已存在此风险

* 升级到 minSdk=31 后，Android 12 设备的 WebView 普遍比 Android 7 更新

* **风险不因 minSdk 升级而增加，反而可能降低**

#### 风险 R-02：color-mix() 玻璃面板效果失效 🟡

**风险描述**：`color-mix()` 需要 WebView 111+。失效时，玻璃面板背景、阴影、表格边框等将变为透明或默认值。

**影响范围**：约 12 处（玻璃面板、浮动光晕、聊天气泡表格）

**当前处理**：用户明确要求**不降级**

**风险评估**：同 R-01，风险不因 minSdk 升级而增加

#### 风险 R-03：dvh 单位在旧 WebView 上失效 🟢

**风险描述**：`dvh` 需要 WebView 108+。失效时，`h-[100dvh]` 将被忽略，容器高度可能异常。

**影响范围**：1 处（[MainLayout.tsx](file:///d:/projects/Mobile-Tavern/src/components/MainLayout.tsx#L66)）

**当前兜底**：已有 `visualViewport.height` 动态设置高度的 JS 兜底逻辑

**风险评估**：低风险，已有充分兜底

#### 风险 R-04：field-sizing 在旧 WebView 上失效 🟢

**风险描述**：`field-sizing: content` 需要 WebView 123+。失效时，textarea 不会自动调整高度。

**影响范围**：1 处（[components/ui/textarea.tsx](file:///d:/projects/Mobile-Tavern/components/ui/textarea.tsx)）

**当前兜底**：已有 `onInput` JS 自动调整高度的兜底逻辑

**风险评估**：低风险，已有充分兜底

#### 风险 R-05：原生版本判断逻辑产生死代码 🟢

**风险描述**：minSdk=31 后，`Build.VERSION.SDK_INT >= Q` (29) 和 `>= R` (30) 判断将永远为 true，产生死代码。

**影响范围**：[AndroidThemeBridge.kt](file:///d:/projects/Mobile-Tavern/src-tauri/plugins/android-bridge/android/src/main/kotlin/com/aitavern/plugin/androidbridge/AndroidThemeBridge.kt) L317/L346/L369

**风险评估**：

* 不影响编译，不影响运行时行为

* 仅存在代码可读性问题

* **遵循用户"不进行任何系统功能降级"原则，保留现有逻辑不动**

### 5.3 风险综合评估

| 风险编号                 | 等级   | 是否因 minSdk 升级而新增 | 是否有兜底        |
| -------------------- | ---- | ---------------- | ------------ |
| R-01 oklch 失效        | 🟡 中 | ❌ 否（既有风险）        | ❌ 无（用户要求不降级） |
| R-02 color-mix 失效    | 🟡 中 | ❌ 否（既有风险）        | ❌ 无（用户要求不降级） |
| R-03 dvh 失效          | 🟢 低 | ❌ 否（既有风险）        | ✅ 有（JS 兜底）   |
| R-04 field-sizing 失效 | 🟢 低 | ❌ 否（既有风险）        | ✅ 有（JS 兜底）   |
| R-05 死代码             | 🟢 低 | ✅ 是（新增）          | ✅ 无影响        |

**总体结论**：minSdk 升级到 31 **不会引入新的兼容性风险**。所有 CSS/JS 特性兼容性问题均为既有风险，与 minSdk 版本无关，而是取决于设备 WebView 的更新状态。Android 12 设备的 WebView 普遍比 Android 7 更新，升级后整体兼容性**预期更好**。
