### 1.2 测试失败与未执行原因分析

#### 1.2.1 test:bridge 失败分析

- **错误信息**：`ENOENT: no such file or directory, open 'D:\projects\Mobile-Tavern\src\utils\tavernHelperBridge.ts'`

  - [bridgeCore.ts](file:///d:/projects/Mobile-Tavern/src/utils/tavernHelper/bridgeCore.ts)
  - [zodMock.ts](file:///d:/projects/Mobile-Tavern/src/utils/tavernHelper/zodMock.ts)
  - [tavernHelperMocks.ts](file:///d:/projects/Mobile-Tavern/src/utils/tavernHelper/tavernHelperMocks.ts)
  - [scriptIframe.ts](file:///d:/projects/Mobile-Tavern/src/utils/tavernHelper/scriptIframe.ts)
  - [mvuParser.ts](file:///d:/projects/Mobile-Tavern/src/utils/tavernHelper/mvuParser.ts)
  - [index.ts](file:///d:/projects/Mobile-Tavern/src/utils/tavernHelper/index.ts)
- **功能覆盖替代**：Bridge 功能已由 Vitest 套件中的 [mvuScriptIntegration.test.ts](file:///d:/projects/Mobile-Tavern/tests/vitest/mvuScriptIntegration.test.ts)（42 项测试）与 [tavernHelperBridgeIntegration.test.ts](file:///d:/projects/Mobile-Tavern/tests/vitest/tavernHelperBridgeIntegration.test.ts) 充分覆盖
- **风险评估**：低。功能层面无回归，仅测试脚本入口未跟随源码重构同步更新
- **修复建议**：将 `run_bridge_tests.cjs` 第 7 行路径更新为 `../src/utils/tavernHelper/bridgeCore.ts`，或废弃该脚本（Vitest 已覆盖）

#### 1.2.2 test:e2e 未执行分析

- **错误原因**：Playwright 浏览器二进制未安装（`%USERPROFILE%\AppData\Local\ms-playwright` 目录不存在）

- **修复建议**：配置 `PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright` 后执行 `npm run test:e2e:install` 安装浏览器，再运行 `npm run test:e2e`

## 二、统一结构化测试套件

### 2.1 测试套件分层架构

```
│  tests/vitest/*.test.ts(x) — React 组件 + DOM 断言          │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 集成测试（自研 Node assert Runner）               │
│  tests/run_all_tests.ts — 内核/服务/管道/安全全链路          │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: 单元测试（独立 .cjs 脚本）                        │
│  tests/*.cjs — Bridge/Zod/Bundle 预处理逻辑验证             │
└─────────────────────────────────────────────────────────────┘
```

## 三、多维度质量评估

### 3.1 评估方法与评分体系

采用**五维度加权评分模型**，每个维度满分 100 分，综合评分按加权平均计算：

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 功能完整性 | 25% | 核心功能覆盖度、P0/P1 修复率、测试通过率 |
| 代码质量 | 20% | 类型安全、架构解耦、单文件行数合规、编码规范 |
| 性能表现 | 20% | 启动加载、长列表渲染、异步资源回收、快速通道 |
| 安全性 | 20% | SSRF/防腐层/加密/原型污染/密钥管理 |
| 用户体验 | 15% | 错误兜底、键盘避让、安全区域、交互流畅度 |

**等级评定标准**：

| 等级 | 分数区间 | 含义 |
|------|---------|------|
| A | 90-100 | 生产就绪，可安全发布 |
| B | 80-89 | 基本就绪，存在可接受的次要缺陷 |
| C | 70-79 | 条件就绪，需修复关键问题后方可发布 |
| D | 60-69 | 未就绪，存在多项阻断性问题 |
| F | <60 | 严重缺陷，不具备发布条件 |

**评估依据**：

| 评估项 | 状态 | 得分 |
