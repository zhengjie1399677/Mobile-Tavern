# 受控浏览器自动化测试规范

> [!IMPORTANT]
> **此文件为 Mobile Tavern 行为指导手册的子规范，定义了白名单/黑名单场景以及网络限制应对细则。**

---

### 1. 允许的使用场景（白名单）
*   **声明式测试脚本**：通过 Playwright（`*.spec.ts`）或 Vitest（`*.test.ts` / `*.test.tsx`）驱动的、纳入仓库版本管理的测试用例，用于：
    *   React 组件渲染测试（组件挂载、Props 驱动行为、DOM 断言）。
    *   端到端集成测试（应用启动、Tab 切换、关键交互流程、MVU 开关门控契约验证）。
*   **本地运行验证**：通过 `npm run test:unit` / `npm run test:e2e` 等聚合脚本一次性批量执行，而非逐条手动驱动浏览器。

### 2. 严禁的使用场景（黑名单）
*   **临时探索性调试与界面观看**：严禁任何一次性、未固化为测试脚本的浏览器自动化调用（例如：起一个浏览器去「看看页面长什么样」「点一下试试」），这是历史上 Token 飞速流失的核心痛点。**特别地，在开发、调试或解决问题时，AI 助手严禁以任何理由调用浏览器子代理（如 browser_subagent）在后台打开页面进行『观看』、『确认 UI 效果』或『手动点击交互』**。
*   **替代单元测试**：E2E 测试不得取代纯逻辑单元测试；能用 Node 侧纯函数断言覆盖的逻辑，必须优先用单元测试，浏览器自动化仅用于验证真实 DOM / 窗口契约。

### 3. 强制技术约束（呼应「开发者网络代理环境限制」）
*   **本地静态化资源**：测试必须使用本地静态资源，严禁加载境外 CDN（Google Fonts、jsdelivr 等），以规避开发者 TUN（虚拟网卡）代理模式下的连接重置与死锁。
*   **超时与重试上限**：浏览器自动化必须配置合理的超时上限（导航 ≤10s、单个断言/定位 ≤5s）与有限重试次数，严禁无界等待导致会话挂起。
*   **浏览器二进制下载**：Playwright 浏览器二进制须配置国内镜像（如设置 `PLAYWRIGHT_DOWNLOAD_HOST` 环境变量）下载，避免境外网络拉取失败。
*   **聚合执行**：测试运行、依赖安装等需在计划批准时通过 `allowedPrompts` 显式授权，不得在未授权状态下擅自拉起浏览器进程。
