### 📝 低风险问题 & 边缘隐患 (Low Severity / P3)

#### 1. 快速高频连击“新建会话”导致物理数据库空会话堆积 (Rapid Multi-click on 'New Session')
*   **严重程度**：⭐ (Low)
*   **涉及模块**：[useChat.tsx](file:///e:/modules/Mobile-Tavern/src/hooks/useChat.tsx) -> `handleStartNewSession`
*   **触发条件**：用户在侧边栏或会话管理界面，快速高频双击或连击“新建会话”按钮。
*   **根本原因**：`handleStartNewSession` 是异步操作，在执行 `await saveSession(newSession)` 并更新 `activeSessionId` 的这几百毫秒延迟中，按钮未被置灰，用户可以进行二次/三次触发。
*   **产生影响**：在物理数据库 IndexedDB 中连续写入多个内容相同、ID 不同的“未命名新会话”，导致会话列表杂乱。

# 🔍 必须手动排查与验证的测试专项 (MANUAL TESTING PROTOCOLS)
# =====================================================================

> [!IMPORTANT]
> 以下专项由于涉及 Tauri 跨端物理特性、二进制 Exif 元数据互通性及物理断网时序，**无法通过自动化单元测试进行模拟**，必须由测试员按照以下步骤进行手动验证并确认：

## 1. Tauri 安卓真机跨域免证书直传测试 (Cross-Origin Direct Fetch Verification)
*   **测试目的**：验证在 Tauri 安卓真机或 Tauri 模拟器打包环境下，`isClientMode()` 能 100% 判定为 `true` 并绕过 CORS 跨域限制直连 OpenAI/LLM API。
*   **手动验证手段**：
    1. 使用 Tauri CLI 打包安卓 APK 并安装至真机。
    2. 打开 App，在“设置”中选择“使用原生直连 (Client Mode)”。
    3. 输入自定义的 API Base URL 与 Key，点击“测试连接”。
    4. 预期结果：即使没有部署本地 Server 的反向代理，安卓端也能成功请求并取得响应，无任何 CORS 跨域红字报错。

## 2. SillyTavern 第三方 PNG 角色卡导入/导出物理兼容性测试 (EXIF Format Compatibility)
*   **测试目的**：保证导出的精美 PNG 角色卡被导出为标准 tEXt Chunk 且 Key 为 `chara`，内容为 Base64 化的标准 SillyTavern JSON，能被任何第三方客户端成功解析。
*   **手动验证手段**：
    1. 在本 Tavern 应用中新建一个复杂角色，并包含 3 条以上的世界书（Lorebook）记录。
    2. 点击“导出精美 PNG 角色卡”下载生成的图片。
    3. 打开第三方主流客户端（如标准网页版 SillyTavern 或 Tavern-Web）。
    4. 将该 PNG 图片直接拖入第三方客户端的“导入角色卡”中。
    5. 预期结果：第三方客户端能 100% 成功识别头像，并正确解构出该角色的名字、人设、开场白、首句以及完整的世界书配置（没有字段遗失或报错）。

## 3. SSE 流式通信物理断网边缘恢复测试 (SSE Network Droppage Recovery)
*   **测试目的**：验证在 AI 疯狂打字流式输出时，如果突遇物理断网、Wi-Fi 切换或网络丢包，系统能否保留已生成的文字并优雅退出。
*   **手动验证手段**：
    1. 发起一条需要超长回复的 Prompt 对话。
    2. 在 AI 开始疯狂打字（流式输出）第 5-10 秒期间，物理拔掉网线或手动关闭设备 Wi-Fi。
    3. 预期结果：
        * 对话框中的 AI 思考提示 `💭...` 被正确清除。
        * 已经生成的半截文本（如前 50 字）被成功保留，并触发 `wasSessionDeleted` 之外的安全存储，完整落盘写入物理 IndexedDB。
        * 发送框的置灰状态（`isSending`）在超时或连接中断时被 `finally` 块 100% 成功释放（可重新发送），界面不卡死。
