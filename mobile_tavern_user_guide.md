# 📱 Mobile Tavern (移动酒馆) 用户操作指南与客服排雷知识库
*Version: 1.5.0*

本知识库专为 Mobile Tavern 客户端的使用者、客服人员及云端大模型客服助理（如“雪团”挂件客服）设计。文档梳理了所有的起步步骤、小白避坑操作、核心界面交互以及高级配置规范。

## 🔰 第一部分：新手起步与纯小白防雷常识

Mobile Tavern 是一款专注于移动端原生体验的角色扮演客户端。为避免用户在使用过程中产生误解或误报 Bug，请务必熟读并遵循以下常识：

### 1. 脑壳与容器分离：软件不是免费聊天 App
> [!IMPORTANT]
> **本软件不提供任何内置的 AI 脑区，也没有免费聊天的功能！**
> 它是一个完全保存在本地的壳子与容器。用户必须自己准备“脑子”（即大模型 API 接口密钥）和“灵魂”（即酒馆角色卡）。

*   **小白避坑指南**：
    *   **获取密钥**：用户需前往第三方大模型平台（例如国内最常用的 DeepSeek 开放平台，或 OpenAI、API 中转站）注册账号并充值，创建一个以 `sk-` 开头的 API Key。
    *   **常规配置路径**：打开 App -> 点击右下角【设置】->【常规】-> 找到【API服务端点配置】。
    *   **Base URL 填写错误**：Base URL 是接口端点，千万不能直接填成官网的网页登录地址！
        *   *错误示例*：`https://deepseek.com`
        *   *正确示例*：`https://api.deepseek.com/v1` （末尾的 `/v1` 不可遗漏）
    *   **验证连接**：填写 API Key 和 Base URL 后，务必点击“获取模型列表”，选择要使用的对话模型（如 `deepseek-chat`），最后点击“测试连接”验证通道是否畅通。

### 2. 离线模型 (Ollama) 配置与跨域 CORS 锁死
部分用户希望在本地电脑上部署离线大模型（如 Ollama），让手机连电脑进行聊天，但经常遇到“连接失败”或“CORS 跨域报错”。

*   **小白避坑指南**：
    *   **局域网 WiFi 检查**：手机与运行 Ollama 的电脑必须连接在“同一个 WiFi 局域网”内。
    *   **IP 填写规则**：Base URL 必须填电脑在局域网中的内网 IP 地址（例如 `http://192.168.1.100:11434`），而不能填电脑本机的 `http://localhost:11434` 或 `http://127.0.0.1:11434`（否则手机 WebView 会将请求发送给自己而导致失败）。
    *   **解锁 CORS 跨域限制**：由于手机 WebView 同源安全策略限制，Ollama 默认会拦截外部跨域请求。**必须在电脑的系统环境变量中，添加名为 `OLLAMA_ORIGINS` 且值为 `*` 的变量，然后彻底重启 Ollama 服务**，否则手机绝对无法成功连接！

### 3. 数据本地私密保存：卸载软件会导致数据彻底清空
> [!WARNING]
> **本软件 100% 离线运行，无任何中央云端服务器！**
> 您的所有角色卡、聊天历史、世界设定集全部物理保存在您手机本地的数据库（IndexedDB）中。

*   **小白避坑指南**：
    *   **卸载警示**：一旦用户卸载 App、清理应用数据、换手机，所有的聊天数据都会被永久擦除，且无法找回。
    *   **备份方案**：用户应当养成定期备份的习惯。在【设置】->【存储】中找到“数据管理”，开启“加密备份”并设置一个牢记的密钥，点击“导出备份”生成 `.json` 文件，并将其保存到云盘或发送到安全位置。

### 4. 导出和下载的文件去哪了？
由于移动端安全机制，WebView 无法像电脑浏览器一样弹出下载对话框让用户选择保存路径。

*   **小白避坑指南**：
    *   **物理存储路径**：当点击“导出备份”、“导出角色 JSON”或在角色馆“保存卡片 PNG 图片”时，软件会调用系统原生桥接写出文件。
    *   **查找方式**：文件会默默地保存至您手机系统公共的 **`/Download`（或“下载”）文件夹** 下。请使用手机自带的“文件管理”或“文件”应用前往该目录下查找，千万不要误以为导出没有反应。

### 5. 导入角色卡提示格式错误
*   **小白避坑指南**：
    *   **必须是原图或 JSON**：用户导入的卡片必须是符合 SillyTavern 规范的 `.json` 数据文件，或者是带 EXIF 元数据的 PNG 图片。
    *   **防踩坑常识**：用户在群聊或贴吧看到的角色卡图片，如果是通过“屏幕截图”、或者是被微信/QQ等软件发送时未勾选“发送原图”而被自动转码压图的 JPG 图片，**图片内部包含的配置代码已经被清除**，这只是普通图片，无法导入。必须获取原图 PNG 文件。

## 🎛️ 第二部分：主界面四大页签交互说明

### 1. 🎭 角色馆 (Characters Directory)
管理角色卡的起步页。
*   **卡片导入**：点击右上角“+”号选择带元数据的 PNG 图片或 JSON 配置文件。
*   **在位编辑（Dossier Editor）**：直接在界面中修改该角色的人设描述（Personality）、首句问候语（First Message）、系统提示词（System Prompt）。
*   **高级视觉覆盖（Visual Settings）**：每个角色都可以开启个性化样式。支持单独覆盖气泡颜色、文本颜色、聊天区底图（支持高斯模糊和透明度），还可以书写自定义 CSS 代码。

### 2. 💬 历史对话 (Chat History)
管理同一个角色的多条故事线。
*   **分支平行宇宙**：支持针对同一个角色开辟多条不同的聊天进程。点击克隆可以从当前的聊天记录复制一个新的分支线，方便探索多重故事结局；点击删除可以彻底抹除废弃分支。

### 3. 📖 世界书 (Worldbook / Lorebook)
通过关键字在聊天时动态激活并注入背景知识的设定集。

> [!CAUTION]
> **🚨 展开机制（小白极易误判点）：**
> 点进“世界书”页面后，默认状态下是折叠的，**界面上只显示各个角色卡的名字（如“雪团”、“莉莉丝”）或“🌎 全局共享词库”的标题，看不见具体的词条列表**。
> **必须再次点击角色名称或全局词库的面板标题，才会向下展开显示其专属的词条列表！** 请勿因为第一眼看见是空的而误以为加载失败。

*   **双轨模式**：
    *   **🌎 全局共享词库**：所有角色聊天时都会检索并激活它。
    *   **👤 角色专属回路**：只有和该角色聊天时才会激活。支持滑动列表一键将专属词库转换为全局共享词库，或反向对调。
*   **词条属性**：
    *   *触发关键词*：英文半角逗号分隔。聊天内容提及这些词时自动载入设定。
    *   *常驻*：开启后，即使没提到关键词，也强制把这块设定塞给 AI 记忆。
    *   *禁用*：打上勾后该词条呈现灰色透明，暂时失效。

### 4. ⚙️ 设置 (System Settings)
系统级的大脑和参数调校控制台。
*   **预设 Sampler**：调节大模型的 Temperature（温度）、Top P（核采样）等参数。
*   **核心指令集**：可以在这里调整 Main System Prompt（底层扮演指令）、Jailbreak（破限提示词，常用于解除大模型机械化说教）、Post-History Instructions（尾部纪律指令，强制大模型严防越狱、禁止替玩家代操作）。
*   **正则脚本过滤器**：支持通过正则表达式（Regex）对 AI 发送的消息或收到的消息执行替换过滤。例如添加正则过滤掉思考标签 `<think>...</think>` 里的深度思考过程。

## 💬 第三部分：对话房交互规则与客服小猫雪团

### 1. 表情包立绘动态差分
如果导入的角色卡内包含多张表情图片（如 angry, blush, sad, joy），并且在设置配置了表情关键字匹配正则：
*   在聊天时，系统会自动扫描大模型回复的文本内容。一旦匹配到情绪关键词，页面顶部的角色立绘会自动渐变切换为对应的差分表情包图片。
*   如果图片缺失或未配对，系统会平滑回退使用默认的主头像（neutral/default），绝对不会抛出异常或显示破碎的图片红叉。

### 2. Markdown 渲染与星号分色
*   默认执行标准 Markdown 排版。
*   如果用户在角色卡视觉设置中勾选了“星号格式化排版 (enableAsteriskFormatting)”，页面上所有被星号 `*` 包裹的非对白描述性文字（如动作描述、心理活动）都会转为低饱和度的柔和灰色斜体，而对白部分保持高亮对比，营造沉浸式小说阅读质感。

### 3. 挂件客服助理小猫（雪团）
小猫常驻于软件底栏页面右下角，它有两套运行逻辑：
*   **本地吐槽（离线，无需网络）**：
    *   *深夜模式*（23:00 - 05:00）：提醒用户熬夜会掉毛。
    *   *挂机发呆*：停留超过 3 分钟没有点击，催促用户找角色聊天或配置 API 密钥。
    *   *API 报错*：系统捕获到 API 失败后，小猫头顶会跳出吐槽气泡并切换为 sleepy 犯困表情。
    *   *戳戳它*：单击播放摸头洗脸动画（relax 状态）。
*   **智能问答与 60s 超时保护**：
    *   双击小猫可弹起对话框。未联网时由本地关键字库兜底提供操作指引；联网时（配置了 FC 直连）会将消息转发给大模型。
    *   **60秒超时保护**：如果用户的网络极度卡顿，请求在 **60秒** 内没有返回，前端会强制中断连接，并以雪团口吻回复超时提示，避免界面锁死。

## 🎨 第四部分：角色卡高级 V2 规范与设定双轨兼容 (针对创作者)

为了在 Mobile Tavern 中获得完美的角色扮演和设定加载效果，可以遵循以下高级卡片编写标准：

### 1. 高级视觉自定义字段 (Visual Settings)
系统支持读取角色卡 JSON 根节点下的 `visualSettings` 字段以自动覆盖全局气泡样式：
*   `bubbleColor`: AI 的对话气泡背景色（Hex 颜色码，如 `#1a1b26`）。
*   `bubbleTextColor`: AI 的对话文本颜色（Hex 颜色码，如 `#a9b1d6`）。
*   `userBubbleColor`: 用户的对话气泡背景色（Hex 颜色码）。
*   `userBubbleTextColor`: 用户的对话文本颜色（Hex 颜色码）。
*   `primaryColor`: 主题高亮色（Hex 颜色码）。
*   `secondaryColor`: 次级辅助色（Hex 颜色码）。
*   `backgroundColor`: 聊天区底色（Hex 颜色码）。
*   `backgroundImageUrl`: 聊天背景图 URL（支持置空或空字符串）。
*   `backgroundOpacity`: 背景图不透明度（0-1 之间的 Float，如 0.2）。
*   `backgroundBlur`: 背景虚化程度（px，如 8）。
*   `enableAsteriskFormatting`: 是否启用星号格式化渲染（Boolean，若为 `true` 则渲染星号为柔和斜体）。
*   `customCss`: 允许注入的自定义 CSS 样式表。

### 2. 世界设定双轨兼容规范 (Lorebook & Character Book)
为了在 Mobile Tavern 中实现最高效的设定加载，建议在编写角色卡时采用**双轨容器兼容结构**。在输出 JSON 角色卡时，应当在根节点同时输出 `lorebookEntries` 和 `character_book`，且两者的 entries 条目数量、结构和内容保持一致。

#### 📚 五个世界书标准 (Five Lorebooks Standard)：
设计设定集时，以**刚好 5 条核心条目**作为行业编写标准：
1. **背景与世界观 (World Background)**: 定义故事所处的宏观世界、地理、历史、种族和规则。触发词为世界名词或概念词。
2. **主角色设定：{{char}} (Character Specs: Char)**: 存储 {{char}} 详细的身世、外貌、性格、习惯和私密偏好。通常设为常驻 (`constant: true`)。
3. **次角色/玩家设定：{{user}} (Character Specs: User)**: 定义对话另一方（玩家）的身份、外貌特征以及双方在故事中的关系纽带。
4. **对话指令与规避规则 (LLM Rules & Jailbreak)**: 防御 LLM 机器人化和机械化，约束其保持语气风格，遵循复杂人性描绘原则。必须设为常驻 (`constant: true`)，注入位置通常为 `in_chat`。
5. **场景剧情与动态关系机制 (Scenario & Relationship)**: 详细规定当前环境以及关系变化的判定规则与指令引导。触发词为当前场景相关的关键词。
